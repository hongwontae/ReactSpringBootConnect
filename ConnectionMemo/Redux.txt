Redux

리덕스 사용하는 방법은 쉽다 => 활용 가이드는 없다. => 아키텍쳐가 중요하다.


Store => 데이터가 있다. 문지기가 있다. 어플리케이션 당 하나만 있다.
Reduxer => 함수다. 올바른 값이 들어오지 않으면 reJect
Action => value 값

Dispatch => 스토어가 변경되면 싹 다 뿌린다.
Selector => 판단한다. 가져올 것 아닐 덧
이 두개는 컴포넌트에서 훅스로 빠진다.

총 3개를 추가해야 한다.

extraReducer => 비동기처리


Store의 문지기
createSlice로 외부에서 호출할 함수를 알려주고 Store에 저장

이제 전역이다. => 전역이니까 사용하는 컴포넌트가 있으면 같이 행동한다.


메뉴를 클릭 => 목록에 띄우게 해보자

payload => 전달되는 데이터를 뜻한다.

slice의 처음은 현재 상태, 두 번은 변경된 상태 => 객체로 줘야한다.

우리는 화면에 나오는 데이터 중에 내부의 데이터, 외부의 데이터를 구분하고 개발을 해야한다.


개발 우선순위 => 이 프로젝트가 무엇이냐?..
